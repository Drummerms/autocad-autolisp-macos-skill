# About 1005 Group Codes (DXF)

1005 xdata group codes have the same behavior and semantics as soft pointers, which means that they are translated whenever
the host object is merged into a different drawing. However, 1005 items are not translated to session-persistent identifiers
or internal entity names in AutoLISP and ObjectARX. They are stored as handles.
