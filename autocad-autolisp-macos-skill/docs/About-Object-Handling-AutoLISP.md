# About Object Handling (AutoLISP)

AutoLISP provides functions for handling objects. The object-handling functions are organized into two categories: functions
that retrieve the entity name of a particular object, and functions that retrieve or modify entity data. See Object-Handling
Functions (AutoLISP) in AutoLISP Function Synopsis (AutoLISP), for a complete list of the object-handling functions.

#### Topics in this section

* [About Accessing an Object’s Entity Name (AutoLISP)](About-Accessing-an-Object’s-Entity-Name-AutoLISP.md)

  An AutoLISP routine must obtain an object’s entity name to make subsequent calls to the entity data or selection set functions.
* [About Entity Data Functions (AutoLISP)](About-Entity-Data-Functions-AutoLISP.md)
* [About Entity Handles and Their Uses (AutoLISP)](About-Entity-Handles-and-Their-Uses-AutoLISP.md)

  The handent function retrieves the name of an entity with a specific handle.
* [About Entity Data Functions and the Graphics Screen (AutoLISP)](About-Entity-Data-Functions-and-the-Graphics-Screen-AutoLISP.md)

  Changes to the drawing made by the entity data functions are reflected on the graphics screen, provided the entity being deleted,
  undeleted, modified, or created is in an area and on a layer that is currently visible.
* [About Non-Graphical Object Handling (AutoLISP)](About-Non-Graphical-Object-Handling-AutoLISP.md)

  A drawing database contains two types of non-graphical objects: dictionary and symbol table objects.
