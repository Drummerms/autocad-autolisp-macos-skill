# About Controlling the Visibility of Objects in Dynamic Blocks

Using visibility states, you can create a block with different graphical representations.

A visibility state is a custom property that allows only specified geometry to display in a block reference. For example,
using visibility states, you can combine the following four weld symbols into a single dynamic block.

![](../images/GUID-99B4D76E-9421-4F0A-8F96-4EE55895FF3E-low.png)

![](../images/GUID-C41B2A43-3E2B-465B-860C-1229007B57A1-low.png)

After you combine the geometry, you add a visibility parameter.

You can then create and name a different visibility state for each weld symbol (for example, WLD1, WLD2, WLD3, and WLD4).
In the following example, the WLD1 visibility state is displayed in the Block Editor. The geometry that is dimmed is invisible
for the WLD1 visibility state.

![](../images/GUID-1A163C66-CAE8-41B0-AD8C-8635F72DFCDB-low.png)

The Block Editor contextual tab displays the name of the current visibility state. This area of the tool bar also provides
several tools for working with visibility states.

When you work with visibility states, you may or may not want to see the geometry that is invisible for a given state. Use
the Visibility Mode button (BMODE) to determine whether geometry is displayed.
