# Include Information With Sheet Sets, Sheets, Subsets, and Views

Sheet sets, sheets, subsets, and views include several types of information. This information, called
*properties*, includes titles, descriptions, file paths, and custom properties that you define and place on a layout with fields.

## Different Properties for Different Levels (Owners)

Sheet sets, sheets, subsets, and views represent different levels of organization, and each includes different types of properties.
You specify the values for these properties after you create the sheet sets, sheets, subsets, or view.

In addition, you can define custom properties for a sheet set and set. The values for custom properties for a sheet set are
typically specific to just that sheet set. For example, a custom property for a sheet set might include the contract number.
The values for custom properties for sheets are typically specific to each sheet. For example, a custom property for a sheet
might include the name of the designer.

You cannot create custom properties for subsets.

## View and Edit Properties

You can view and edit the properties of a sheet set, sheet, subset, or view at the bottom of the Sheet Set Manager. The properties
and values that are displayed depend on what you select. You can edit the value of a property by clicking its value.

## Place Sheet Set Information on a Layout

Information about a sheet set, or the sheets and views in a sheet set, can be inserted as a field onto a layout that a sheet
set references. When the information in a sheet set is changed, the fields that reference the sheet set are updated to reflect
the latest value when the layout is saved, printed, or published.

Placeholder fields are used to reference sheet set information in a drawing template or block. When a drawing template or
block containing a placeholder field is added to a sheet set or sheet, the field value is resolved. If the field value of
a placeholder field is not resolved, the field displays
*####* as its current value.
