# About Data Types (AutoLISP)

AutoLISP expressions are processed according to the order and data type of the code within the parentheses. Before you can
fully utilize AutoLISP, you must understand the differences among the data types and how to use them.

#### Topics in this section

* [About Integers (AutoLISP)](About-Integers-AutoLISP.md)

  Integers are whole numbers; numbers that do not contain a decimal point.
* [About Reals (AutoLISP)](About-Reals-AutoLISP.md)

  A real is a number containing a decimal point. Numbers between -1 and 1 must contain a leading zero.
* [About Strings (AutoLISP)](About-Strings-AutoLISP.md)
* [About Lists (AutoLISP)](About-Lists-AutoLISP.md)

  A list is a group of related values separated by spaces and enclosed in parentheses.
* [About Selection Sets (AutoLISP)](About-Selection-Sets-AutoLISP.md)
* [About Entity Names (AutoLISP)](About-Entity-Names-AutoLISP.md)

  An entity name is a numeric label assigned to objects in a drawing.
* [About VLA-objects (AutoLISP/ActiveX)](About-VLA-objects-AutoLISP-ActiveX.md)

  Objects in a drawing can be represented as ActiveX (VLA) objects.
* [About File Descriptors (AutoLISP)](About-File-Descriptors-AutoLISP.md)

  A file descriptor is a pointer to a file opened by the AutoLISP open function.
* [About Symbols and Variables (AutoLISP)](About-Symbols-and-Variables-AutoLISP.md)

  AutoLISP uses symbols to refer to functions and data holders.
